# What agents should not do

Agents must not:

- Write `HappyDesigns`; use `happydesigns`.
- Put business logic in brand layers.
- Put brand-specific code in `ui`.
- Duplicate Nuxt, Nuxt UI, Docus, or Nuxt Studio docs.
- Scatter raw Tailwind colors and `dark:` variants through components.
- Use UI visibility as authorization.
- Expose private Business OS details in public docs unless they are intended as durable docs.
- Publish `.agents/` scratch state as documentation.
- Treat current package descriptors as runtime extension loading approval.
- Treat development or demo API boundaries as production authorization.
- Turn customer help into technical developer docs.
- Over-document unstable implementation details.

## If unsure

Stop and identify the product boundary. Then use the architecture docs and MCP to confirm the current rule before making the change.
