Docus and Nuxt Content expose LLM-friendly content files so agents can retrieve compact documentation without scraping rendered pages.

LLM files make the docs easier for AI tools to index and cite. They complement MCP and skills; they do not replace either.

Use LLM files to discover and read documentation. Use MCP when a tool supports interactive retrieval. Use skills for behavior.

## Expected outputs

- `/llms.txt` for a compact index.
- `/llms-full.txt` for broader content.
- Raw Markdown routes where Nuxt Content exposes them.
- Docus MCP for structured discovery.

## Authoring implications

Write pages so they make sense as standalone Markdown. Avoid relying on visual layout alone to carry meaning. Tables, headings, diagrams, and contextual links should still communicate in raw form.

## Boundary

LLM files are not a secret or private knowledge base. Do not publish private implementation details, temporary plans, credentials, customer data, or unstable task logs.
