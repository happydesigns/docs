Docus serves skills from the root `skills/` directory at `/.well-known/skills/`.

Skills should stay short and point agents back to the docs. Do not copy the whole documentation set into a skill.

## Initial skills

```txt
skills/
  happydesigns-architecture/
    SKILL.md

  happydesigns-identity/
    SKILL.md

  happydesigns-nuxt-ui/
    SKILL.md
```

## Skill roles

| Skill | Use it for |
| --- | --- |
| `happydesigns-architecture` | Modifying or creating happydesigns projects, packages, layers, docs, or product architecture. |
| `happydesigns-identity` | Creating, modifying, or applying identity and brand layers. |
| `happydesigns-nuxt-ui` | Styling or composing Nuxt UI components inside happydesigns projects. |

Keep `SKILL.md` files focused. Move detailed material into docs pages or small references only when needed.
