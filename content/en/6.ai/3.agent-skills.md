Docus serves skills from the root `skills/` directory at `/.well-known/skills/`.

Skills stay short and point agents back to the docs. Do not copy the whole documentation set into a skill.

Skills teach agents how to act while MCP and LLM files help agents retrieve what is true.

Create a skill when agents need a repeatable workflow, not when they only need another documentation page.

## Published skills

| Skill | Use it for |
| --- | --- |
| `happydesigns-architecture` | Modifying projects, packages, layers, modules, docs, or product architecture. |
| `happydesigns-identity` | Creating, modifying, or applying identity and brand layers. |
| `happydesigns-nuxt-ui` | Styling or composing Nuxt UI components inside happydesigns projects. |
| `happydesigns-docs` | Writing or restructuring Docus documentation for happydesigns. |

## Skill boundary

Keep `SKILL.md` concise. Put durable explanation in docs. Add supporting references only for detail that is too large or situational for the default skill body.
