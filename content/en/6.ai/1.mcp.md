Docus exposes a built-in MCP server at `/mcp`. MCP means Model Context Protocol; in these docs, it is the tool interface agents use to read published documentation.

Agents use MCP to read published docs instead of guessing from stale memory.

Use MCP first when an agent needs to know what the published docs say about architecture or product ownership.

Use MCP to answer what the docs say. Use skills to decide how to act on that information.

## Reading model

```txt
MCP = read published documentation
Skills = workflow, constraints, and behavior
```

## Use MCP for

- Finding available docs pages.
- Reading published architecture guidance.
- Checking product boundaries before editing.
- Confirming identity and Nuxt UI policy.

## Use skills for

- Activating the right workflow.
- Applying constraints.
- Remembering how to act while changing code.
