# Where code belongs

Use this table when a change could fit in more than one place.

| Change | Belongs in |
| --- | --- |
| New reusable content card pattern | `ui` |
| happydesigns logo component | `brand` / identity layer |
| Client logo and client colors | Client identity layer |
| Customer homepage text | Customer site content |
| Invoice finalization logic | Business OS service or domain package |
| App shell navigation policy | App or product layer, maybe later module |
| Nuxt UI color mapping | Identity or brand layer |
| Customer-specific dashboard widget | Nuxt layer or reviewed extension point |
| n8n integration behavior | Business OS API, webhook, or integration module |

## Placement rule

Put the most stable, reusable part at the lowest useful level. Put project-specific composition at the project level. Do not move code into a reusable package only because it might be useful later.
