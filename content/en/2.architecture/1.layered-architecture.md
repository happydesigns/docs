happydesigns separates reusable foundations, installable behavior, identity, and specific project work.

Layering lets products share structure without sharing the wrong responsibilities. It also gives agents a reliable map before changing code.

## General model

| Layer | Owns | Avoid putting here |
| --- | --- | --- |
| Plain TypeScript | Types, rules, calculations, parsing, validation, predictable services, and contracts. | Request context, presentation, deployment bindings. |
| Nitro | HTTP handlers, server plugins, jobs, adapters, deployment bindings, and runtime context. | Reusable domain rules or UI composition. |
| Nuxt modules | Installable behavior, hooks, generated files, typed integration, and runtime config. | Brand expression or project-specific pages. |
| Nuxt layers | UI, branding, pages, layouts, components, app config, and content conventions. | Business rules shared by several runtime channels. |
| Host app or project | The final product experience and route-level composition. | Shared product logic that belongs lower. |

## Website and docs model

1. `happydesigns/ui` provides reusable interface and content foundations.
2. Identity and brand layers provide tokens, logos, app config, and visual expression.
3. The site or product project composes content, pages, routes, and final experience.

## Business OS model

Business behavior lives below every runtime channel. A runtime channel is an entry point such as UI, API, MCP, a scheduled job, or a webhook.

1. Domain packages own entities, policies, business rules, and documents.
2. Application services own commands, queries, workflows, and coordination.
3. Runtime channels expose the same services through UI, API, MCP, jobs, and webhooks.
4. The Nuxt UI app provides the human operations interface.
5. Brand and customer layers change presentation only.

## Placement rule

Choose the lowest layer that can own the change without hurting product quality. For channel-specific placement examples, read [Runtime placement](/en/architecture/runtime-placement).
