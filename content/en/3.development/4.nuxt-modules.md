Nuxt modules are used for installable behavior and integration points.

Use modules when a behavior should be installed into Nuxt apps with explicit hooks, runtime config, generated files, server handlers, or typed contracts.

Modules wrap stable integration contracts. They should not expose private app internals by accident.

## Use modules for

- Registering hooks.
- Adding server routes.
- Defining runtime config.
- Generating files.
- Generating types.
- Adding server handlers.
- Providing behavior that should be reusable across Nuxt apps.

## Examples

- `happydesigns/nuxt-variants` belongs here because it installs typed layout variant behavior.
- Customer-specific behavior can become a module if it is a reviewed extension point.
- Business OS integrations should use modules or server contracts, not brand layers.
