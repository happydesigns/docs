Nuxt modules are used for installable behavior and integration points.

## Use modules for

- Registering hooks.
- Adding server routes.
- Installing runtime config.
- Generating files and types.
- Wiring stable integrations.
- Providing behavior that should be reusable across Nuxt apps.

## Module rule

A module should wrap a stable contract. It should not reach into private app internals and make them part of an accidental extension API.

If the behavior is deterministic domain logic, keep that logic in a plain TypeScript package and let the module install the Nuxt integration around it.
