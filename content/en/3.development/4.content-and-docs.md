Docus is used for stable human and agent documentation.

Docs turn stable decisions into reusable knowledge. They are not a place where every task note, implementation scratchpad, or customer tutorial is published.

Publish stable operating knowledge. Keep temporary work state outside published docs.

## Publishing standard

Published docs are stable and reader-facing:

- Reader-facing.
- Stable enough to guide repeated work.
- Useful to both humans and agents.
- Focused on product, architecture, workflow, and stable decisions.

They are not a public scratchpad:

- Temporary implementation logs.
- `.agents/` scratch work.
- Unreviewed planning notes.
- Raw task state.
- A full copy of external product docs.

Business OS can store knowledge-base registry metadata around Docus spaces. That does not make operational scratch notes public documentation.

## Product docs

Product-specific deep docs belong on dedicated product documentation sites. This docs site explains how those products fit together.
