# Product map

| Product / repo | Meaning | Role |
| --- | --- | --- |
| `happydesigns/ui` | Interface foundation | Reusable Nuxt layer with Nuxt UI, Nuxt Content, content schemas, H-prefixed components, app config conventions, and Studio readiness. |
| `happydesigns/identity` | Identity system | Product for the generic identity and brand-layer system. The short ecosystem name may still be `id`. |
| `happydesigns/brand` | Current concrete brand layer | Existing happydesigns brand layer with colors, tokens, logo primitives, typography, Docus styling, and visual QA playground. |
| `happydesigns/business` | Business OS | Automation-first operating system for individuals, small companies, clubs, and agent-powered companies. |
| `happydesigns/nuxt-variants` | Layout and config variants | Reusable Nuxt module for shared layouts that behave differently per page through typed variants. |
| `happydesigns/docs` | Ecosystem docs | This Docus site. |
| `happydesigns/help` | Customer help | Future non-technical help center for customers using their websites. |

## Naming direction

The short names `ui`, `id`, and `os` are intentional ecosystem shorthand.

`identity` / `id` should be treated as the generic identity system direction. `brand` currently exists as the concrete happydesigns brand implementation. Do not collapse those concepts into one thing.
