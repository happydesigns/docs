# Ecosystem map

| Area | Product | Purpose |
| --- | --- | --- |
| Interface | `ui` | Reusable UI and content foundation. |
| Identity | `id` / `brand` | Brand and identity layers. |
| Operations | `business` / Business OS | Workflows, tasks, knowledge, agents, and business records. |
| Docs | `docs` | Ecosystem docs and AI source of truth. |
| Help | `help` | Future customer-facing guides. |
| Variants | `nuxt-variants` | Typed page and layout variant system. |

## Relationship

```txt
docs.happydesigns.de
  = central knowledge and agent entrypoint

ui
  = reusable interface layer

id / brand
  = identity and brand expression layers

business
  = Business OS product and workflow system

help
  = customer-facing usage docs, not developer architecture
```

The docs site should help contributors decide where durable knowledge belongs. It should not absorb every product's deep reference material.
