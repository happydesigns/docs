The docs site connects the product docs, packages, and customer help site.

## Ecosystem diagram

Read the diagram from `docs` outward. The docs site sets shared operating rules, while product repositories contain implementation detail. `ui` provides the interface foundation, `id` defines reusable identity systems, `brand` applies happydesigns expression, `os` owns Business OS workflows, and `help` is the customer-facing docs site.

```mermaid
flowchart TD
  Docs["docs<br/>ecosystem operating manual"] --> UI["ui<br/>interface foundation"]
  Docs --> Identity["id<br/>Identity"]
  Docs --> Brand["brand<br/>happydesigns brand layer"]
  Docs --> Business["os<br/>Business OS"]
  Docs --> Variants["nuxt-variants<br/>layout/config variants"]
  Docs --> Help["help<br/>customer help center"]

  UI --> Brand
  UI --> Identity
  Identity --> Brand
  UI --> Business
  Variants --> UI
  Business --> Help
```

## Product detail

Use the diagram for product relationships. Use the [product map](/en/products/product-map) for exact product keys, packages, repositories, and docs links.

## Boundary

The docs site explains how the pieces fit together. It does not absorb implementation guides from `ui`, `id`, `os`, or `nuxt-variants`.
