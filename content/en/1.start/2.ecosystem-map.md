The ecosystem has one shared strategy layer and several product-specific implementation layers.

## Ecosystem diagram

Read the diagram from `docs` outward. The docs site sets shared operating rules, while product repositories own implementation. `ui` provides the interface foundation, `id` owns reusable identity systems, `brand` applies happydesigns expression, `os` owns Business OS workflows, and `help` is the customer-facing docs site.

```mermaid
flowchart TD
  Docs["docs<br/>ecosystem operating manual"] --> UI["ui<br/>interface foundation"]
  Docs --> Identity["id<br/>Identity"]
  Docs --> Brand["brand<br/>happydesigns brand layer"]
  Docs --> Business["os<br/>Business OS"]
  Docs --> Variants["nuxt-variants<br/>layout/config variants"]
  Docs --> Help["help<br/>customer help center"]

  UI --> Brand
  UI --> Identity
  Identity --> Brand
  UI --> Business
  Variants --> UI
  Business --> Help
```

## Product detail

Use the diagram for relationships between products. Use the [product map](/en/products/product-map) for the canonical product list, package identities, and product documentation boundaries.

## Boundary

The docs site explains how the pieces fit together. It does not absorb implementation guides from `ui`, `id`, `os`, or `nuxt-variants`.
