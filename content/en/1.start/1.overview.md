happydesigns builds reusable web systems with Nuxt, Nuxt UI, Nuxt Content, Docus, and agent-assisted development workflows.

These docs are the ecosystem operating manual. They explain the shared product map, architecture model, identity strategy, documentation rules, and AI workflow that guide work across repositories.

## When to use this site

Use this site when a change crosses product boundaries or when an agent needs published rules before editing code. The docs answer:

- Which product owns this concern?
- Which layer contains the change?
- Which source of truth does an agent read first?
- Which details belong in dedicated product docs or customer help docs?

If a topic affects how happydesigns products fit together, document it here. If it explains how to use one product in depth, document it in that product's dedicated docs. If it teaches customers how to edit or operate their site, it belongs in customer help docs.

That boundary keeps this site useful as an operating manual instead of turning it into a mixed archive of strategy, reference, and support content.

## Scope

This site owns shared operating knowledge:

- Product boundaries and naming.
- Shared architecture principles.
- Identity and brand-layer policy.
- Development and Docus authoring rules.
- AI, MCP, LLM, and skill workflow.

It intentionally does not own product manuals or customer support material:

- Full product API references.
- Customer help content.
- Temporary `.agents/` planning notes.
- Private implementation logs.
- Copies of Nuxt, Docus, Nuxt Content, or Nuxt UI docs.
