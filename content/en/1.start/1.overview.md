happydesigns builds reusable web systems with Nuxt, Nuxt UI, Nuxt Content, Docus, and agent-assisted development workflows.

These docs are the ecosystem operating manual. They explain the shared product map, architecture model, identity strategy, documentation rules, and AI workflow that should guide work across repositories.

## Purpose

Use this site when a change crosses product boundaries or when an agent needs current rules before editing code. The docs should answer:

- Which product owns this concern?
- Which layer should contain the change?
- Which source of truth should an agent read first?
- Which details belong in future product docs or customer help docs?

## Decision rule

If a topic affects how happydesigns products fit together, document it here. If it explains how to use one product in depth, document it in that product's dedicated docs. If it teaches customers how to edit or operate their site, it belongs in future help docs.

## This site owns

- Product boundaries and naming.
- Shared architecture principles.
- Identity and brand-layer policy.
- Development and Docus authoring rules.
- AI, MCP, LLM, and skill workflow.

## This site does not own

- Full product API references.
- Customer help content.
- Temporary `.agents/` planning notes.
- Private implementation logs.
- Copies of Nuxt, Docus, Nuxt Content, or Nuxt UI docs.
