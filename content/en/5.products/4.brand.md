`brand` is the happydesigns-specific brand layer. It applies the happydesigns visual system above reusable UI foundations.

Treat `brand` as the happydesigns implementation, not the reusable identity product.

## Product scope

The brand layer owns the happydesigns expression:

- happydesigns color roles.
- Tailwind theme tokens.
- Nuxt UI color-role mappings.
- CSS variables.
- Typography defaults.
- Logo and brand primitives.
- Docus styling and visual QA pages.

Domain behavior, customer-specific business logic, authorization, audit, API behavior, MCP tools, webhooks, production credentials, and financial finalization belong outside the brand layer.
