`brand` is the concrete happydesigns brand layer. It applies the happydesigns visual system above reusable UI foundations.

Treat `brand` as a concrete implementation, not the generic identity product.

## Links

- [Documentation](https://brand.happydesigns.de)
- [GitHub](https://github.com/happydesigns/brand)
- [npm](https://www.npmjs.com/package/@happydesigns/brand)

## Product scope

The brand layer owns the concrete happydesigns expression:

- happydesigns color roles.
- Tailwind theme tokens.
- Nuxt UI semantic mappings.
- CSS variables.
- Typography defaults.
- Logo and brand primitives.
- Docus styling and visual QA surfaces.

Domain behavior, customer-specific business logic, authorization, audit, API behavior, MCP tools, webhooks, production credentials, and financial finalization belong outside the brand layer.
