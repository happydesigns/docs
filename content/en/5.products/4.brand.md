`happydesigns/brand` publishes the private package `@happydesigns/brand`.

## Purpose

`brand` is the concrete happydesigns brand layer. It applies the happydesigns visual system above reusable UI foundations.

## Decision rule

Treat `brand` as current implementation, not the generic identity product.

## Owns

- happydesigns color roles.
- Tailwind theme tokens.
- Nuxt UI semantic mappings.
- CSS variables.
- Typography defaults.
- Logo and brand primitives.
- Docus styling and visual QA surfaces.

## Does not own

Domain behavior, customer-specific business logic, authorization, audit, API behavior, MCP tools, webhooks, production credentials, or financial finalization.
