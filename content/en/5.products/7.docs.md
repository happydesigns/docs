`happydesigns/docs` is this Docus site.

It is the central source of truth for happydesigns ecosystem strategy and the default AI entrypoint for architecture, development principles, identity strategy, and product boundaries.

## Purpose

Use this site when a decision affects more than one product or when agents need current operating rules.

## Decision rule

Docs should remain durable, reader-facing, high-signal, and retrievable by humans and agents.

## What it provides

- Human-readable documentation.
- AI-readable content.
- Docus MCP at `/mcp`.
- LLM files such as `/llms.txt`.
- Installable agent skills from root `skills/`.

## Boundary

It should not become a customer help center, raw task log, or full product reference for every package.
