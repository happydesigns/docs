`happydesigns/help` is the customer-facing help center.

Help docs should explain how customers use and maintain their websites. They should not teach internal architecture.

Write help docs for non-technical users. Keep developer strategy in `docs`.

## Help topics

- How to edit website content.
- How to use Nuxt Studio.
- How to publish changes.
- How to manage images.
- SEO basics.
- Customer-facing website workflows.

## Boundary

`help` is not where architecture docs, product internals, reusable package details, or agent workflow rules belong.
