`happydesigns/ui` publishes `@happydesigns/ui`.

It is the reusable interface foundation for happydesigns websites and apps.

## Purpose

Use `ui` when a project needs shared Nuxt UI structure, content conventions, components, and app config foundations.

## Decision rule

`ui` provides reusable interface infrastructure. It is not a brand layer.

## What it provides

- Nuxt 4 layer foundation.
- Nuxt UI 4.
- Nuxt Content 3.
- Nuxt Studio readiness.
- H-prefixed components such as `<HFooterButtons />`.
- App config conventions.
- Content collections and schemas.

## Registered modules

- `@nuxt/ui`
- `@nuxt/content`
- `nuxt-content-traits`
- `nuxt-studio`
- `@nuxt/eslint`

## Content collections

| Collection | Type | Source | Purpose |
| --- | --- | --- | --- |
| `page` | page | `pages/**/*.{md,yaml}` | Normal website pages. |
| `snippet` | page | `snippets/**/*.{md,yaml}` | Reusable snippets. |
| `article` | page | `articles/**/*.{md,yaml}` | Articles with traits. |
| `event` | page | `events/**/*.{md,yaml}` | Events with traits. |
| `user` | data | `users/**/*.{md,yaml}` | People, authors, and users. |

## What it is not

`ui` is not a brand layer. It may contain UI defaults and reusable structure, but brand-specific expression belongs above it in identity and brand layers.
