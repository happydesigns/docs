# `id` / `identity`

`id` is the short ecosystem name for the `happydesigns/identity` product.

The product should define how brand layers are created, structured, documented, and applied on top of reusable Nuxt UI foundations.

## Scope in this docs site

This page is only an ecosystem overview. It explains how identity relates to `ui`, `brand`, and client layers.

The dedicated `happydesigns/identity` docs should own implementation guides, APIs, package usage, templates, token schemas, examples, migrations, and product reference material.

## Relationship to `brand`

`brand` currently exists as the concrete happydesigns brand layer. `id` is the broader product direction for creating reusable identity systems for happydesigns and clients.

```txt
ui
  = reusable interface foundation

id / identity
  = generic identity / brand-layer product

brand
  = current concrete happydesigns brand implementation
```

Client identities should use the same identity model without forking UI or product behavior.
