# `docs`

`happydesigns/docs` is this Docus site.

It is the central source of truth for the happydesigns ecosystem and the default AI entrypoint for architecture, development principles, identity strategy, and product boundaries.

## What it provides

- Human-readable documentation.
- AI-readable content.
- Docus MCP at `/mcp`.
- `llms.txt`.
- Installable agent skills from root `skills/`.

## Boundary

Docs should remain durable, reader-facing, and high-signal. They should not become a customer help center, raw task log, or full product reference for every package.
