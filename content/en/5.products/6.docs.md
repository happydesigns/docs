`docs` is this Docus site. It explains how happydesigns products fit together and gives agents the same architecture rules that humans read.

Use this site when a decision affects more than one product or when an agent needs the published rules before changing code.

Keep this site focused. Product implementation details, customer help, and temporary task notes belong elsewhere.

## What it provides

- Human-readable documentation.
- AI-readable content.
- Docus MCP at `/mcp`.
- LLM files such as `/llms.txt`.
- Installable agent skills from root `skills/`.
