# What is `id`?

`id` is the short ecosystem name for the happydesigns identity system.

In these docs, identity means the strategy for how brand layers are created, structured, documented, and applied on top of reusable Nuxt UI foundations.

```txt
ui = interface foundation
id = identity / brand-layer system
os = operations / Business OS
```

## Product direction

`happydesigns/identity` should be the product that turns this strategy into implementation: packages, templates, scaffolding, token schemas, examples, and reference docs.

This site should only document the ecosystem boundary and core policy. The dedicated identity docs should explain how to build with the product.

`id` is not just the happydesigns logo or current brand implementation. The current concrete implementation is `happydesigns/brand`.
