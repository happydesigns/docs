# Brand layers

A brand layer adapts presentation on top of reusable UI and product foundations.

## A brand layer may provide

- Tailwind theme tokens.
- Nuxt UI semantic color mappings.
- CSS variables.
- Typography defaults.
- Logos and favicons.
- App metadata.
- Layout wrappers.
- Page and frame motifs.
- Brand primitives.
- Component default variants.
- Docs styling.
- Client-specific navigation labels.
- Legal links and footer content.

## A brand layer must not provide

- Business or domain logic.
- Authorization.
- Invoice numbering.
- E-Rechnung generation.
- Audit behavior.
- Tenant isolation.
- Storage behavior.
- API behavior.
- MCP tools.
- Webhook behavior.
- Production credentials.
- Financial finalization.
