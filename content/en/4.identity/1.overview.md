Identity is the strategy for applying brand expression on top of reusable product foundations without moving product behavior into brand layers.

## Purpose

This section explains the ecosystem-level boundary. The dedicated `happydesigns/identity` product docs should later own implementation guides, APIs, templates, token schemas, examples, migrations, and reference material.

## Decision rule

`identity` is the generic system. `brand` is the current concrete happydesigns brand layer.

```mermaid
flowchart LR
  UI["ui<br/>interface foundation"] --> Identity["identity<br/>generic brand-layer system"]
  Identity --> Brand["brand<br/>happydesigns implementation"]
  Identity --> Client["client identity layers<br/>customer expression"]
  Brand --> Projects["docs, sites, product experiences"]
  Client --> Projects
```

## This section owns

- Identity and brand-layer boundaries.
- Relationship between `identity`, `brand`, `ui`, and clients.
- Token flow and Nuxt UI semantic policy.
- Current happydesigns brand usage rules.

## This section does not own

- Full identity package APIs.
- Token schema reference docs.
- Brand asset downloads.
- Customer-specific brand manuals.
