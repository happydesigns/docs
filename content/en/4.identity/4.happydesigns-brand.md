`happydesigns/brand` is the current concrete happydesigns brand layer. It publishes the private package `@happydesigns/brand`.

It is not the final generic identity system. Treat it as the current happydesigns brand implementation that may later inform `identity`.

## Purpose

This page documents the current brand layer at the ecosystem level. It is not a full component or asset reference.

## Decision rule

Use the brand layer for happydesigns expression. Do not move its concrete visual identity into `ui`.

## What it provides

- Tailwind v4 theme tokens.
- Nuxt UI semantic color mappings.
- Warm CSS variables.
- Typography defaults.
- Logo and brand primitives.
- Frame and grid primitives.
- Docus styling.
- Visual QA playground.

## Semantic color mapping

| Nuxt UI semantic color | Brand palette |
| --- | --- |
| `primary` | `coral` |
| `secondary` | `petrol` |
| `success` | `seafoam` |
| `info` | `plum` |
| `warning` | `butter` |
| `error` | `rose` |
| `neutral` | `sand` |

Peach is intentionally not mapped as a Nuxt UI semantic color.

## Color roles

| Role | Meaning |
| --- | --- |
| `coral` | Signature accent and logo dot. |
| `graphite` | Premium text and filled-control color. |
| `warm-white` | Default page ground. |
| `peach` | Atmosphere and logo-field color. |
| `petrol` | Technical counterweight for developer, module, and system signals. |
| `butter` | Soft warning. |
| `plum` | Quiet information. |
| `seafoam` | Constructive success and sustainability. |
| `sand` | Warm neutral scale. |
| `rose` | Restrained destructive or error signal. |

## Usage rules

- Use graphite for premium filled actions.
- Use coral for focus rings, dots, active states, and precise signature details.
- Do not turn coral into the default large CTA surface everywhere.
- Use peach for atmosphere and controlled brand fields, not as a global semantic color.
- Let structure carry the system: frames, fine borders, quiet grids, and warm neutrals.
- Keep artistic gestures in controlled canvas areas, not global wallpaper.
