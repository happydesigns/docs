# Identity

Identity documentation explains how brand expression sits above reusable UI foundations.

This section is ecosystem-level guidance. It defines identity boundaries, naming, and policy for happydesigns docs and agent workflows.

The dedicated `happydesigns/identity` product docs should own implementation guides, package APIs, templates, token schemas, scaffolding workflows, examples, migrations, and versioned reference material.
