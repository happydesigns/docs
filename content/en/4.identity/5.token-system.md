happydesigns styling flows from central identity tokens into component usage.

The token system keeps light/dark behavior, brand colors, and component styling centralized.

Components use Nuxt UI color utilities. Identity layers define the palette and map it to named roles.

## Token flow

| Stage | Owns | Example |
| --- | --- | --- |
| Raw palette | Brand colors defined by the identity or brand layer. | `coral`, `petrol`, `seafoam`, `plum`, `butter`, `rose`, `sand` |
| Color roles | Nuxt UI roles mapped from the palette. | `primary`, `secondary`, `success`, `info`, `warning`, `error`, `neutral` |
| CSS variables | Shared interface variables for backgrounds, text, borders, radius, and accents. | `--ui-primary`, `--ui-bg`, `--ui-text`, `--ui-border` |
| Component defaults | Central Nuxt UI slot, variant, and default behavior. | buttons, cards, forms, tabs, alerts |
| Component usage | Local component markup that uses Nuxt UI color utilities. | `text-default`, `bg-elevated`, `border-muted` |

## Component rule

Components usually use Nuxt UI color utilities such as `text-default`, `bg-elevated`, and `border-muted`. They do not choose raw brand colors unless they are part of the identity layer, a logo, or a controlled brand canvas.
