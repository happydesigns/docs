Client identities should be separate Nuxt layers built on the same identity system.

## Purpose

Client identities let customer projects look and read like the client while still using the same product foundation.

## Decision rule

Create a client identity layer before forking base UI or product logic.

## A client identity layer should contain

- Tokens.
- Logos.
- Metadata.
- Component defaults.
- Brand primitives.
- Presentation-specific copy.

## A client identity layer should not

- Fork the base UI foundation.
- Own product logic.
- Change authorization.
- Change financial behavior.
- Change tenant isolation.
- Add production integration behavior without a reviewed contract.

Customer-specific behavior needs a Nuxt module or stable extension contract, not a brand layer.

## Examples

- A client logo belongs in the client identity layer.
- Customer homepage copy belongs in the customer project.
- Customer-specific dashboard behavior belongs in a reviewed extension point.
