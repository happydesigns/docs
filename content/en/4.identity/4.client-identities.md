Client identities are separate Nuxt layers built on the same identity system. They let customer projects look and read like the client while still using the same product foundation.

Create a client identity layer before forking base UI or product logic.

## Layer scope

A client identity layer contains client-specific presentation:

- Tokens.
- Logos.
- Metadata.
- Component defaults.
- Brand primitives.
- Presentation-specific copy.

A client identity layer does not:

- Fork the base UI foundation.
- Own product logic.
- Change permissions.
- Change workflows or data rules.
- Add production integration behavior.

If a customer needs different workflow logic, keep it in the customer project or in the product that owns the workflow. The client identity layer can theme and compose the experience, but it does not own the workflow.

## Examples

- A client logo belongs in the client identity layer.
- Customer homepage copy belongs in the customer project.
- Customer-specific dashboard layout belongs in the customer project or client Nuxt layer.
- Reusable dashboard logic belongs in the product that owns the dashboard.
